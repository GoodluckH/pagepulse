Need to detect website changes programmatically? PagePulse provides a free REST API that lets you monitor any webpage and receive webhook notifications when content changes. No credit card required.

Why Use a Website Monitoring API?

Building change detection from scratch is harder than it looks:

  • Infrastructure overhead - You need reliable cron jobs, queue systems, and redundancy
  • Anti-bot challenges - Many sites block scrapers; you need rotating proxies and browser fingerprinting
  • Content parsing - Extracting meaningful changes from messy HTML is complex
  • Scaling issues - Monitoring thousands of pages requires significant resources

PagePulse handles all of this so you can focus on your core product.

Quick Start

Start monitoring a URL in one API call:

curl -X POST https://api.pagepulse.app/monitors \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "webhook": "https://your-app.com/webhook",
    "interval": 60
  }'

That's it! PagePulse will check the page every 60 minutes and POST to your webhook when changes are detected.

Use Case: Price Drop Alerts

Build a price monitoring feature for your e-commerce app:

// Node.js example
const axios = require('axios');

// Create monitor for product price
async function watchPrice(productUrl, userWebhook) {
  const response = await axios.post('https://api.pagepulse.app/monitors', {
    url: productUrl,
    webhook: userWebhook,
    selector: '.product-price', // Only monitor price element
    interval: 30
  });
  
  return response.data.monitor_id;
}

// Your webhook handler
app.post('/price-webhook', (req, res) => {
  const { url, diff_summary } = req.body;
  
  // Notify user about price change
  sendPushNotification(user, `Price changed! ${diff_summary}`);
  
  res.status(200).send('OK');
});

Error Handling

The API returns standard HTTP status codes:

  • 200 - Success
  • 400 - Bad request (invalid parameters)
  • 401 - Unauthorized (invalid API key)
  • 429 - Rate limit exceeded
  • 500 - Server error

Error responses include a message:

{
  "error": "rate_limit_exceeded",
  "message": "You've exceeded 100 API calls today. Upgrade to Pro for higher limits.",
  "retry_after": 3600
}
